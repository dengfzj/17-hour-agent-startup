# 2026-06-14 更新：本地 API-only 开发模式与中英文 UI 切换

语言：**中文** | [English](./update-2026-06-14-en.md)

记录日期：2026-06-14

## 摘要

本次更新主要做三件事：明确本地开发时前端与 API 的边界，增加应用内中英文全局切换，并刷新公开文档，确保 README 里的启动方法与当前脚本一致。

## 更新内容

### 1. 本地 API 模式改为显式 API-only

现在 `npm run api` 会以 API-only 模式启动后端：

```bash
npm run api
```

默认 API 地址：

```text
http://127.0.0.1:8787
```

健康检查地址：

```text
http://127.0.0.1:8787/api/health
```

在本地 API-only 模式下，直接打开 `http://127.0.0.1:8787/` 返回 API 404 页面是预期行为。前端开发入口仍然是 Vite 的 5173 端口。

### 2. Vite 前端开发服务使用 API 代理

`npm run dev` 会启动前端：

```text
http://127.0.0.1:5173
```

Vite 配置会把 `/api/*` 请求代理到本地 Express API：`http://127.0.0.1:8787`。因此开发环境下前端代码可以继续使用同源 `/api` 路径。

### 3. 推荐的完整本地开发命令

日常本地开发推荐使用：

```bash
npm run dev:full
```

它会同时启动：

- Vite 前端：`http://127.0.0.1:5173`
- Express API：`http://127.0.0.1:8787/api/health`

### 4. 生产风格单服务模式仍然保留

如果要按生产风格在本地运行：

```bash
npm run build
npm run start
```

这种模式下，后端会从 `dist/` 托管已构建的前端，同时暴露 API。这个行为与 `npm run api` 分开：`npm run api` 在本地开发时只提供 API。

### 5. 应用内中英文全局切换

应用现在支持全局语言切换：

- 主工作台页面在顶部栏显示语言按钮；
- 公开 checkout、legal、onboarding、recovery 页面显示固定语言按钮；
- 语言偏好保存在 `localStorage`；
- `html lang` 和浏览器标题会跟随语言变化；
- 用户输入的业务数据、CSV 内容、邮箱、URL 和数字金额不会被本地化层改写。

实现上继续使用 UTF-8 源文件，并依赖 `index.html` 中的 UTF-8 声明，避免中文乱码。
